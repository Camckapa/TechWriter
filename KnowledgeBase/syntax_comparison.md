Сравнительная таблица синтаксиса для языков разметки **DITA**, **Markdown**, **reStructuredText (reST)** и **AsciiDoc**. Таблица охватывает элементы, которые используются в проекте.

Ключевое различие, которое стоит иметь в виду: **DITA** — это XML-ориентированный язык, где используются семантические теги (например, `<cmdname>` для команд, `<filepath>` для пути к файлу). **Markdown**, **reST** и **AsciiDoc** — это облегченные языки разметки, текст размечается с помощью специальных символов.

| Элемент | **DITA** | **Markdown (md)** | **reStructuredText (reST)** | **AsciiDoc (adoc)** |
| :--- | :--- | :--- | :--- | :--- |
| **Курсив** | `<i>курсив</i>` | `*курсив*` или `_курсив_` | `*курсив*` | `_курсив_` |
| **Жирный текст** | `<b>жирный</b>` | `**жирный**` или `__жирный__` | `**жирный**` | `*жирный*` |
| **Заголовки** | `<title>Заголовок темы</title>` | `# H1`, `## H2`, `### H3` и т.д. | `Заголовок 1`<br>`==================`<br><br>`Заголовок 2`<br>`------------------` | `= Заголовок 1`<br>`== Заголовок 2`<br>`=== Заголовок 3` |
| **Параграфы** | `<p>Текст параграфа</p>` | Текст, отделенный пустой строкой от других блоков. | Текст, отделенный пустой строкой от других блоков. | Текст, отделенный пустой строкой от других блоков. |
| **Нумерованные списки** | `<ol>`<br>`<li>Элемент 1</li>`<br>`<li>Элемент 2</li>`<br>`</ol>` | `1. Элемент 1`<br>`2. Элемент 2` | `1. Элемент 1`<br>`2. Элемент 2` | `. Элемент 1`<br>`. Элемент 2` |
| **Ненумерованные списки** | `<ul>`<br>`<li>Элемент 1</li>`<br>`<li>Элемент 2</li>`<br>`</ul>` | `- Элемент 1`<br>`- Элемент 2`<br>или `*` | `- Элемент 1`<br>`- Элемент 2`<br>или `*` | `* Элемент 1`<br>`* Элемент 2` |
| **Примечания (цитаты)** | `<note>Текст примечания</note>` | `> Текст цитаты` | `.. note:: Текст примечания. Отбивается тремя пробелами с новой строки` | `NOTE: Текст примечания` |
| **Таблицы** | `<table>`<br>`<tgroup cols="2">`<br>`<thead>...</thead>`<br>`<tbody>...</tbody>`<br>`</tgroup>`<br>`</table>` | [Синтаксис таблиц в md](#cинтаксис-таблиц-в-markdown) |  [Синтаксис таблиц в reST](#cинтаксис-таблиц-в-rest) | [Синтаксис таблиц в adoc](#cинтаксис-таблиц-в-adoc) |
| **Рисунки (изображения)** | `<image href="путь/к/файлу.png" placement="break"/>` или `<fig>` для подписи | `![Alt-текст](путь/к/файлу.png)` | `.. image:: путь/к/файлу.png` | `image::путь/к/файлу.png[Alt-текст]` |
| **Ссылки** | `<xref href="topic.dita" format="dita"/>` для внутренних и `<xref href="https://example.com" format="html" scope="external"/>` для внешних  | `[Текст ссылки](https://example.com)` | `Текст ссылки <https://example.com>`_<br>Или: `Внешняя ссылка`_ с определением внизу. | `https://example.com[Текст ссылки]` |

## 🔑 Ключевые особенности и различия

*   **Философия DITA**: DITA нацелена на семантическую разметку. Вместо того чтобы просто сделать текст жирным (`<b>`), вы можете использовать `<cmdname>` для команды или `<filepath>` для пути к файлу. Это обеспечивает единообразие, упрощает автоматическую обработку и многократное использование контента, но требует специализированного редактора, такого как Oxygen XML Editor.

*   **Markdown: простота и фрагментация**: Markdown невероятно прост для начала, его синтаксис интуитивно понятен, а исходный код очень читаем. Однако его главный недостаток — отсутствие единого стандарта. Для сложных элементов, таких как таблицы или примечания, приходится использовать разные «диалекты» (например, GitHub Flavored Markdown) или вставлять HTML-код.

*   **reStructuredText: мощь и строгость**: reST — более мощный и строгий язык, чем Markdown. Он тесно связан с экосистемой Python и широко используется с Sphinx для создания документации. Его синтаксис, особенно для таблиц, может быть громоздким и строгим в плане отступов, что создает более высокий порог входа.

*   **AsciiDoc: баланс мощности и читаемости**: AsciiDoc часто рассматривается как «правильный» Markdown. Он предлагает краткий, читаемый синтаксис, но при этом обладает всеми необходимыми для технической документации функциями (сложные таблицы, примечания, перекрестные ссылки, вставка фрагментов) «из коробки», без необходимости в разных диалектах или HTML. AsciiDoc также поддерживает множество форматов вывода (HTML, PDF, DocBook) и стандартизирован, что делает его надежным выбором для больших проектов.
