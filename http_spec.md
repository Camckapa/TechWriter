# Описание протокола HTTP

*HTTP* — протокол прикладного уровня, который предназначен для обмена произвольными данными в интернете. Протокол применим в рамках архитектуры с взаимодействием "клиент-сервер", в которой:

- клиент — инициирует запросы к серверу;
- сервер — хранит ресурсы, обрабатывает запросы от клиента и возвращает ответ.

Сервер выполняет над ресурсом определенную операцию — метод, который явно задан в запросе.

Правила и форматы протокола описаны в спецификациях RFC. На момент написания статьи последней версией протокола является версия 2.0.

HTTP может также использоваться как транспорт для других протоколов прикладного уровня, например, в веб-приложениях.

## Модель взаимодействия

Клиент и сервер обмениваются HTTP-сообщениями по схеме "запрос-ответ". Для идентификации ресурсов в запросе используются глобальные URI. Прокси — промежуточные узлы между клиентом и сервером, которые выполняют транспортную функцию.

Протокол HTTP не предусматривает сохранение состояния запроса, но это может быть реализовано специальными механизмами на стороне клиента и сервера.

## Структура HTTP-сообщений

HTTP-сообщение состоит из [стартовой строки](#стартовая-строка), [заголовков](#заголовки) и [тела сообщения](#тело-сообщения).

### Стартовая строка

Определяет тип сообщения. Формат стартовой строки различается для запроса и ответа.

Формат стартовой строки **в запросе** включает:

1. **Метод** — указывает серверу на то, какая операция должна быть выполнена над ресурсом. Протокол поддерживает следующие методы:

   - `GET` — запрос ресурса.
   - `POST` — передача данных для обработки (например, отправка формы).
   - `PUT` — загрузка данных для замены или создания ресурса.
   - `DELETE` — удаление указанного ресурса.
   - `CONNECT` — создание TCP/IP-туннеля через прокси.
   - `OPTIONS` — получение списка методов, которые поддерживает сервер.
   - `TRACE` — получение данных с учетом информации, которую добавляют или изменяют промежуточные узлы.
   - `PATCH` — частичное изменение ресурса.
   - `HEAD` — аналогичен GET, но возвращает только заголовки (без тела ответа).

2. **Путь к запрашиваемому ресурсу** . Задается в виде унифицированного идентификатора ресурса (URI).

3. **Версию HTTP**.

Пример стартовой строки запроса:

```
GET /wiki/HTTP HTTP/1.0
```

Формат стартовой строки **в ответе** включает:

1. **Версию HTTP**. Как в запросе.
2. **Код состояния**. Трехзначный код, который указывает на результат обработки запроса. Коды состояния сгруппированы по классам:

   - `1xx` (Информационные): Запрос получен, обработка продолжается.
   - `2xx` (Успех): Запрос успешно выполнен.
   - `3xx` (Перенаправление): Для завершения запроса требуется дополнительное действие.
   - `4xx` (Ошибка клиента): Запрос содержит ошибку.
   - `5xx` (Ошибка сервера): Сервер не смог выполнить запрос.

3. **Пояснение**. Текстовое короткое пояснение к коду ответа для пользователя. Является необязательным.

Пример стартовой строки ответа:

```
HTTP/1.0 200 OK
```

### Заголовки

Заголовки — строки, которые позволяют клиенту и серверу обмениваться дополнительными метаданными. Каждый заголовок — отдельная строка, содержащая пару `Ключ: Значение`. Все заголовки, необходимые для взаимодействия клиента и сервера по протоколу HTTP, определены в спецификации RFC. Для дополнительных заголовков используется префикс `X-`.

Заголовки должны отделяться от тела сообщения хотя бы одной пустой строкой.

Пример заголовков:

```
Server: Apache/2.2.11 (Win32) PHP/5.3.0
Last-Modified: Sat, 16 Jan 2010 21:16:42 GMT
X-Request-ID: 123e4567-e89b-41d4-a756-426614174000
```

### Тело сообщения

Используется для передачи данных, связанных с запросом или ответом. Может отсутствовать.

**В запросе:** Тело содержится в сообщении, если это предполагается в методе запроса (например, в методе POST передаются данные для обработки).

**В ответе:** Тело сообщения может отсутствовать. За наличие тела сообщения в ответе отвечают метод и код состояния. Тело сообщения отсутствует:

   - в ответе на запрос с методом `HEAD`;
   - в ответе на запрос с кодом состояния `1xx`, `204` и `304`.

Все другие ответы содержат тело сообщения, даже если оно имеет нулевую длину.
