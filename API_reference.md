# Games
Управление играми

<a id="Add_Games"></a>
## Создание новой игры

**POST /games**

REQUEST BODY SCHEMA: `application/json`
|Параметр|Тип|Обязательность|Описание|
|--|--|--|--|
|player1Id|string|Да|Идентификатор первого игрока (инициатора игры) в формате UUID. Пример: 05e795d2-fba4-402a-9cf2-ef92432dae9b.|
|player2Id|string|Нет|Идентификатор второго игрока. Если не задан, то игра в статусе *Ожидание второго игрока*.|

Пример:

```
{
  "player1Id": "05e795d2-fba4-402a-9cf2-ef92432dae9b",
  "player2Id": "236c8f73-e9fd-4c78-be07-9f26e316a88f"
}
```

Responses:
|Параметр|Описание|
|--|--|
|code|Код ответа. Возможные значения: 201, 400.|
|RESPONSE SCHEMA|Схема ответа.|

RESPONSE SCHEMA: `application/json`

**Код ответа 201**
|Параметр|Тип|Обязательность|Описание|
|--|--|--|--|
|gameId|string|Нет|Идентификатор игры в формате UUID. Пример: 3f0eee32-8eac-4668-9b38-e3e5dc21d607.|
|status|string|Нет|Статус игры. Возможные значения: <br/>`waiting_for_player2` — *Ожидание второго игрока*. <br/>`placement` — *Оба игрока расставляют корабли*. <br/>`active` — *Идет игра*. <br/>`finished` — *Игра завершена*. |
|currentTurn|string|Нет|<!-- ИИ не пояснил, что имел в виду. Информация требует уточнения -->|

Пример:

```
{
  "gameId": "3f0eee32-8eac-4668-9b38-e3e5dc21d607",
  "status": "waiting_for_player2",
  "currentTurn": "2ecb9c19-b5ed-48f1-9fb4-27f408ef0195"
}
```

**Код ответа 400**
|Параметр|Тип|Обязательность|Описание|
|--|--|--|--|
|error|string|Нет|Мнемоника ошибки.|
|code|string|Нет|Код ошибки.|
|details|string|Нет|Детали ошибки.|

Пример:

```
{
  "error": "string",
  "code": "string",
  "details": "string"
}
```
