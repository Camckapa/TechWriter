Описание протокола *HTTP*
=========================

*HTTP* — протокол прикладного уровня, который предназначен для обмена произвольными данными в интернете. Протокол применим в рамках архитектуры с взаимодействием клиент-сервер, в которой:

- клиент — инициирует запросы к серверу;
- сервер — хранит ресурсы, обрабатывает запросы от клиента и возвращает ответ.

Фундаментальные правила и форматы протокола описаны в спецификациях RFC. На момент написания статьи самой последней версией протокола является версия 2.0.

*HTTP* может также использоваться как транспорт для других протоколов прикладного уровня, например, в веб-приложениях.

Модель взаимодействия
---------------------
Клиент, сервер, прокси. Установка соединения. Обмен сообщениями.

В качестве промежуточных узлов между клиентом и сервером выступает прокси. Прокси выполняет функции кеширования, фильтрации балансировки нагрузки или аутентификации.



Структура HTTP-сообщений
------------------------

Каждое HTTP-сообщение состоит из:

1. `Стартовой строки <starting_line>`__
2. `Заголовков <headers>`__
3. `Тела сообщения <message_body>`__

.. _starting_line:

Стартовая строка
~~~~~~~~~~~~~~~~

Определяет тип сообщения и в запросе указывает на конкретный ресурс сервера. Содержимое стартовой строки различается для запроса и ответа.

- Формат стартовой строки в запросе включает:
   - Метод.
   - Путь к запрашиваемому ресурсу.
   - Версия HTTP.
- Формат стартовой строки в ответе включает:
   - Версия HTTP.
   - Код состояния.
   - Пояснение.

.. headers:

Заголовки
~~~~~~~~~

Заголовки — строки в HTTP-сообщении, которые в формате ``Ключ: Значение``.

Заголовки должны отделяться от тела сообщения хотя бы одной пустой строкой.

.. _method:

HTTP-методы
-----------

Метод указывает серверу на то, какая операция должна быть выполнена над ресурсом, который указан в URI.

OPTIONS
~~~~~~~

Получение информации о поддерживаемых методах и возможностях сервера для конкретного ресурса.

GET
~~~

Запрос содержимого ресурса.

HEAD
~~~~

Запрос только заголовков ресурса. В ответе отсутствует тело. 

POST
~~~~

Передача данных на сервер для обработки.

PUT
~~~

Загрузка содержимого запроса по указанному URI.

PATCH
~~~~~

Частичные изменения ресурса.

DELETE
~~~~~~

Удаление ресурса

TRACE
~~~~~

Получение данных с учетом информации, которую добавляют или изменяют промежуточные серверы в запросе.

CONNECT
~~~~~~~

Преобразование соединения запроса в прозрачный TCP/IP-туннель.
